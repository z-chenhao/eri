# Security Policy

Eri is in the `0.x` phase. Security fixes target current `main`; historical-version support is not promised before the first stable release.

Do not disclose vulnerability details, credentials, personal data, or working exploit code in public Issues, Discussions, Pull Requests, logs, or datasets. Use the repository's **Security -> Report a vulnerability** private reporting flow.

Include the affected version or commit, a minimal reproducer, actual impact, attack prerequisites, and suggested mitigation. Do not access data that is not yours, expand side effects, or disclose publicly before coordination.

Maintainers will acknowledge the report, reproduce it, assign severity, and coordinate a fix. Before release, they will check logs, Memory, Episodes, datasets, and build artifacts for residual sensitive data.
