# Travel and life planning global search

This is a planning method, not a user intake questionnaire. Resolve fields from the conversation, reliable memory, device context, research, or safe reversible defaults first. If the request contains a likely typo or ambiguous event, confirm only that interpretation before asking for downstream trip details. Ask at most one smallest material question in each user turn; never request the whole field list at once.

1. Capture travelers, origin catchment, destination flexibility, feasible date range, duration, leave cost, comfort limits, baggage, room needs, budget, and cancellation tolerance over the course of the task.
2. Search a matrix rather than one query: multiple departure/return windows, nearby airports, nonstop and acceptable connections, hotel neighborhoods, and package combinations.
3. Compare total trip cost, door-to-door time, schedule comfort, leave consumed, transfer risk, refundability, location, and experience. Filter cheap but plainly unusable options.
4. Retain a small Pareto set and recommend one option. Include price/time freshness and the strongest tradeoff; never present a sampled search as a mathematical global optimum.
5. Recheck live price and terms before commitment. The user enters credentials and completes first payment; financial and booking commitments require exact strong approval and a real receipt.
